## 目的与范围

本文档描述 Xagent 中的嵌套智能体能力，它通过父子关系实现层次化智能体执行，用于复杂任务分解。嵌套智能体允许智能体生成子智能体，这些子智能体可以半自主运行同时向父级汇报。

关于更广泛的智能体执行生命周期信息，请参阅[核心智能体系统](https://deepwiki.com/xorbitsai/xagent/3-core-agent-system)。关于决定何时创建嵌套智能体的规划引擎详情，请参阅[规划引擎](https://deepwiki.com/xorbitsai/xagent/3.2-planning-engine)。关于可在嵌套智能体内使用的执行模式，请参阅[执行模式](https://deepwiki.com/xorbitsai/xagent/3.3-execution-patterns)。

**源码参考：** [AGENTS.md 8](https://github.com/xorbitsai/xagent/blob/4078c3a0/AGENTS.md?plain=1#L8-L8)，[CLAUDE.md 8](https://github.com/xorbitsai/xagent/blob/4078c3a0/CLAUDE.md?plain=1#L8-L8)

## 概述

嵌套智能体提供层次化结构，父智能体可将子任务委托给子智能体。每个子智能体使用自己的执行上下文、工具和规划周期运行，但继承父级的某些配置和约束。此架构实现复杂任务分解，不同子任务可能需要不同执行策略或专门能力。

嵌套智能体系统在 [src/xagent/core/agent.py](https://github.com/xorbitsai/xagent/blob/4078c3a0/src/xagent/core/agent.py) 的 `Agent` 类中实现，提供生成和管理带父子关系的子智能体的内置支持。

## 父子关系

### 层次结构

Xagent 中的父子关系遵循以下原则：

| 方面 | 父智能体 | 子智能体 |
| --- | --- | --- |
| **创建** | 通过执行运行时生成子智能体 | 任务分解期间由父级创建 |
| **执行上下文** | 有自己的 `AgentContext` | 继承配置但有隔离上下文 |
| **工具访问** | 完全访问已注册工具 | 从父级继承工具访问 |
| **规划** | 独立规划周期 | 独立规划周期 |
| **状态管理** | 跟踪子执行状态 | 管理自己的执行状态 |
| **结果** | 接收并聚合子结果 | 向父级返回结果 |
| **历史** | 维护包括子调用的执行历史 | 维护自己的执行历史 |

### 上下文继承

当父智能体生成子智能体时，某些元素被继承而其他保持隔离：

**继承元素：**
- LLM 模型配置
- 工具注册表和可用工具
- 存储管理器引用
- 可观测性配置（Langfuse 追踪）
- 全局约束和安全参数

**隔离元素：**
- 执行上下文和状态
- 工作内存和对话历史
- 任务特定工作空间目录
- 中间结果和产物

## 创建嵌套智能体

### 通过规划自动创建

创建嵌套智能体最常见的方式是通过规划引擎的任务分解过程。当规划器确定子任务足够复杂值得独立执行时，它可以决定生成子智能体而非执行简单工具。

**图：嵌套智能体创建决策流**

**源码参考：** [src/xagent/core/pattern/](https://github.com/xorbitsai/xagent/blob/4078c3a0/src/xagent/core/pattern/)，[src/xagent/core/runner.py](https://github.com/xorbitsai/xagent/blob/4078c3a0/src/xagent/core/runner.py)

### 智能体生成过程

执行运行时通过以下步骤协调子智能体的生成：

1. **任务分析**：父智能体的规划引擎分析当前任务并识别子任务
2. **复杂度评估**：规划器确定子任务是否需要自己的智能体或可由工具处理
3. **智能体初始化**：如需要子智能体，运行时创建以子任务为目标的新 `Agent` 实例
4. **上下文设置**：子智能体接收继承配置和隔离执行上下文
5. **执行**：子智能体运行自己的规划和执行循环
6. **结果收集**：父智能体接收子的结果并将其合并到自己的执行中

**源码参考：** [src/xagent/core/agent.py](https://github.com/xorbitsai/xagent/blob/4078c3a0/src/xagent/core/agent.py)，[src/xagent/core/runner.py](https://github.com/xorbitsai/xagent/blob/4078c3a0/src/xagent/core/runner.py)

### 执行隔离

每个子智能体在几个关键方面与父级隔离运行：

| 隔离方面 | 实现 |
| --- | --- |
| **工作空间** | 子智能体通过 `workspace.py` 接收隔离工作空间目录 |
| **内存** | 独立对话历史和工作内存 |
| **规划** | 使用 `pattern/` 中模式的独立规划周期 |
| **工具执行** | 工具在子上下文中用子的工作空间执行 |
| **状态管理** | 在存储中跟踪的独立执行状态 |
| **错误处理** | 子错误不崩溃父级；被处理并报告 |

此隔离确保：
- 子智能体无法意外修改父状态
- 多个子智能体可并发执行无干扰
- 子智能体失败被包含并报告
- 每个智能体维护干净执行环境

### 执行历史

父智能体维护包括子智能体调用记录的执行历史。`Agent` 类跟踪：
- 子智能体何时生成
- 分配给每个子的子任务
- 每个子返回的结果
- 执行时间和资源使用
- 子引发的任何错误或异常

此历史支持：
- 复杂多智能体执行的调试和故障排除
- 追踪完整执行树
- 分析智能体性能和决策
- 执行流的重放和重建

## 用例和模式

### 复杂任务分解

嵌套智能体特别适合自然分解为独立子任务的任务：

**示例：研究报告生成**
- **父智能体**：协调整体报告创建
    - **子智能体 1**：研究主题 A（网络搜索、文档分析）
    - **子智能体 2**：研究主题 B（网络搜索、文档分析）
    - **子智能体 3**：综合发现并格式化报告

**示例：多步数据处理**
- **父智能体**：管理数据管道
    - **子智能体 1**：数据收集和验证
    - **子智能体 2**：数据转换和清洗
    - **子智能体 3**：分析和可视化

### 专用智能体角色

不同子智能体可使用不同执行模式或工具集：

| 模式 | 用例 | 智能体配置 |
| --- | --- | --- |
| **ReAct 模式** | 交互式问题解决 | 高推理、试错 |
| **DAG 模式** | 结构化工作流执行 | 预定义步骤、清晰依赖 |
| **工具密集型** | 数据处理任务 | 计算器、文件操作、Python 执行器 |
| **搜索密集型** | 信息收集 | 网络搜索、爬虫、文档解析器 |

**源码参考：** [src/xagent/core/pattern/](https://github.com/xorbitsai/xagent/blob/4078c3a0/src/xagent/core/pattern/)

### 并行子任务执行

虽然当前实现可能顺序执行子智能体，嵌套智能体架构支持未来并行执行，多个子智能体同时处理独立子任务。

## 与直接工具使用比较

理解何时使用嵌套智能体而非直接工具调用对高效执行重要：

| 标准 | 嵌套智能体 | 直接工具 |
| --- | --- | --- |
| **复杂度** | 多步骤、需要规划 | 单步骤、定义明确 |
| **决策制定** | 需要自适应推理 | 简单输入-输出 |
| **状态管理** | 步骤间需要上下文 | 无状态操作 |
| **错误处理** | 可能需要恢复策略 | 简单成功/失败 |
| **资源使用** | 较高（LLM 调用、上下文） | 较低（直接执行） |
| **典型时长** | 分钟到小时 | 秒到分钟 |

**何时使用嵌套智能体：**
- 子任务需要多次 LLM 交互
- 决策点依赖中间结果
- 子任务受益于独立规划
- 不同执行模式对子任务最优

**何时使用直接工具：**
- 任务定义明确，有清晰输入/输出
- 无需中间决策
- 快速执行关键
- 资源效率重要

## 实现细节

### 智能体类结构

[src/xagent/core/agent.py](https://github.com/xorbitsai/xagent/blob/4078c3a0/src/xagent/core/agent.py) 中的 `Agent` 类用以下关键能力实现嵌套智能体支持：
- **父引用**：子智能体维护对其父的引用
- **执行历史**：跟踪所有子智能体生成和结果
- **上下文管理**：管理隔离执行上下文
- **结果聚合**：收集和处理子智能体结果

### 运行时编排

[src/xagent/core/runner.py](https://github.com/xorbitsai/xagent/blob/4078c3a0/src/xagent/core/runner.py) 中的执行运行时编排嵌套智能体创建和执行：
- **智能体工厂**：用正确配置创建新智能体实例
- **生命周期管理**：处理初始化、执行和清理
- **结果收集**：从子智能体收集结果
- **错误传播**：向父报告子智能体错误
